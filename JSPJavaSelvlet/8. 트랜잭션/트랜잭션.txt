Transaction(트랜잭션)
- 여러 작업들을 하나의 작업으로 묶은 작업단위
A -> B 1000원 계좌이체
1. A계좌에서 -1000
에러
2. B계좌에서 +1000

1과2를 하나의 작업으로 묶음
중간에서 에러가 나더라도 원상태로 돌아감


트랜잭션 처리 순서
1. autocommit을 false 지정
2. 작업(쿼리)...
3. commit (커밋)
3-1. 작업중 에러발생하면 롤백

* 트랜잭션이 걸리면 해당 테이블 락(lock)이 걸림
* mysql, mariadb 에서 테이블 생성 시 innodb로 생성
